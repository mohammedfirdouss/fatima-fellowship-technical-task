# Qwen3.5-4B-Base Blind Spots Dataset

A curated collection of 12 diverse prompts where the base language model
[`Qwen/Qwen3.5-4B-Base`](https://huggingface.co/Qwen/Qwen3.5-4B-Base)
produces incorrect or unreliable outputs. Each record contains the input prompt,
the correct expected output, and the model's actual output.

## Model Tested

| Field | Value |
|-------|-------|
| Model | [`Qwen/Qwen3.5-4B-Base`](https://huggingface.co/Qwen/Qwen3.5-4B-Base) |
| Type | Pre-trained base model (not instruction-tuned) |
| Parameters | ~4B |
| Architecture | Hybrid: Gated DeltaNet linear attention + sparse MoE + gated attention + vision encoder |
| Context length | 262,144 tokens |
| Released | February 2026 |
| License | Apache 2.0 |

## How the Model Was Loaded

The model was loaded using the Hugging Face `transformers` library in a Google Colab
notebook (GPU runtime: T4). The full notebook is available in the accompanying
repository.

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

MODEL_NAME = "Qwen/Qwen3.5-4B-Base"

tokenizer = AutoTokenizer.from_pretrained(MODEL_NAME)
model = AutoModelForCausalLM.from_pretrained(
    MODEL_NAME,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)
model.eval()

def generate(prompt: str, max_new_tokens: int = 80) -> str:
    inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
    with torch.no_grad():
        output_ids = model.generate(
            **inputs,
            max_new_tokens=max_new_tokens,
            do_sample=False,
            repetition_penalty=1.1,
            pad_token_id=tokenizer.eos_token_id,
        )
    new_tokens = output_ids[0][inputs["input_ids"].shape[1]:]
    return tokenizer.decode(new_tokens, skip_special_tokens=True).strip()
```

Because this is a **base** (pre-trained only) model, prompts were formatted as
plain Q&A continuations rather than chat templates. The model was run with
greedy decoding (`do_sample=False`) to make outputs deterministic and reproducible.

An alternative Modal runner (`modal_runner.py`) is also provided for GPU access
via the Modal cloud platform.

## Key Observations

Across these 12 probes, Qwen3.5-4B-Base exhibits several recurring failure patterns:

1. **Arithmetic hallucination** — The model produces fluent-looking but numerically
   incorrect answers for multi-digit multiplication. It appears to interpolate from
   training examples rather than compute step-by-step.

2. **Character-level blindness** — Tokenization fragments words into subword pieces,
   making it unreliable for character-counting tasks. The model answers "2" for the
   number of 'r's in "strawberry" despite the correct answer being 3.

3. **Intuitive override failure** — The model defaults to the salient "fast" answer
   on Cognitive Reflection Test items ($0.10 instead of $0.05) rather than performing
   the algebraic reasoning needed.

4. **Base-rate neglect** — On Bayesian probability problems, the model ignores the
   prior (1 in 1000) and anchors entirely on the test accuracy (99%), a classic
   human reasoning bias it has absorbed from training data.

5. **Factual myth absorption** — The model asserts "True" for the Great Wall of China
   space-visibility myth, reflecting the prevalence of this misconception in its
   training corpus.

6. **Linguistic illusions** — The Moses Illusion substitution goes undetected; the
   model answers the question as asked rather than correcting the false premise.

## What Fine-Tuning Dataset Would Fix These Errors?

These errors stem from at least three distinct root causes, and fixing them likely
requires separate fine-tuning interventions:

### 1. Reasoning / Chain-of-Thought Data

Categories affected: arithmetic, cognitive_reflection, spatial_reasoning,
temporal_arithmetic, probabilistic_reasoning, multi_hop_ordering, formal_logic.

The model needs training examples that demonstrate **step-by-step intermediate
reasoning** rather than jumping to a final answer. High-quality datasets:

- **OpenMathInstruct-2** — math word problems with detailed solutions
- **NuminaMath-CoT** — competition mathematics with chain-of-thought
- **LogiQA / ReClor** — formal logical reasoning
- **ProntoQA** — synthetic proof-of-concept logical entailment chains
- Synthetic step-by-step reasoning traces generated with a capable teacher model
  (e.g., distilling GPT-4o or Claude reasoning traces onto simpler math problems)

### 2. Factual Correction / Misconception Debunking Data

Categories affected: factual_misconception, linguistic_illusion.

The model requires exposure to authoritative corrections of common myths, alongside
the false claims themselves so it learns to contrast and reject them. Sources:

- **FactCheck** and **Snopes** articles converted into structured Q&A pairs
- Wikipedia "common misconceptions" page parsed into correction triples
  (myth → correction → source)
- Synthetic contrast pairs: (false claim, "This is false because...")

### 3. Character-Level and Token-Agnostic Tasks

Categories affected: character_level_reasoning, winograd_schema.

Tokenization causes the model to "see" words as chunks rather than characters.
Fine-tuning on explicit character-counting tasks and entity-level annotation
tasks can partially mitigate this. Sources:

- Synthetic character-counting examples across diverse vocabularies
- **Winogrande** dataset — large-scale Winograd schemas for coreference
- Spelling and character-manipulation tasks (anagrams, reversals, vowel counting)

### 4. Commonsense Physics

Categories affected: commonsense_physics.

The model needs training on scenarios where the physically correct answer diverges
from naive arithmetic. Sources:

- **Physical Intuition QA** (custom synthetic dataset)
- **PIQA** — Physical Intuition Question Answering
- Carefully authored counterintuitive physics vignettes with explanations

## How Big a Dataset Would Be Needed?

Based on prior fine-tuning research:

| Error Type | Estimated Examples Needed | Rationale |
|---|---|---|
| Chain-of-thought reasoning | 10,000–50,000 | Skill transfer is broad; a few thousand high-quality CoT traces can significantly improve multi-step reasoning in models of this size (see Magister et al., 2023) |
| Factual misconception correction | 2,000–5,000 | Targeted; a relatively small set of high-quality myth-vs-fact contrast pairs can update specific factual beliefs |
| Character-level tasks | 5,000–20,000 | Requires many examples across different word lengths and letter positions to override tokenization bias |
| Commonsense physics | 3,000–10,000 | Moderate diversity needed to cover physical scenarios |

A combined fine-tuning set of **~20,000–50,000 diverse, high-quality examples**
covering all four categories is a reasonable starting point for a 4B-parameter model.
Instruction tuning at this scale with well-curated data (prioritizing quality over
quantity) is likely to yield meaningful improvements without catastrophic forgetting.

For the mathematical reasoning failures specifically, distillation from a larger
teacher model (generating step-by-step solutions) has been shown to be particularly
effective for small models in the 1-7B range.

## How to Assemble Such a Dataset

1. **Existing benchmarks** — Directly use or reformat LogiQA, Winogrande, PIQA,
   OpenMathInstruct, and NuminaMath (all open-licensed).
2. **Synthetic generation** — Use a capable teacher model (Claude, GPT-4o) to
   generate step-by-step reasoning traces for math and logic problems, then filter
   by correctness.
3. **Myth-correction scraping** — Parse Snopes, Wikipedia "misconceptions", and
   science fact-check articles; write a classifier to extract (false claim, correct
   answer, explanation) triples.
4. **Programmatic generation** — Write Python scripts to generate unlimited
   character-counting, modular-arithmetic, and transitive-ordering examples with
   guaranteed-correct labels (no human annotation needed).
5. **Human annotation** — For the subtler commonsense physics cases, a small pool
   of annotators reviewing synthetic drafts is the most reliable quality gate.
